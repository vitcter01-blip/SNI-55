# SNI Monitoring Panel

Telegram-бот для распределённого мониторинга SNI через удалённые серверы.

## Структура проекта

```
sni_monitor/
├── install.sh              ← установка в один клик (Ubuntu/Debian)
│
├── bot/                    ← центральный узел
│   ├── bot.py              ← Telegram-бот (aiogram 3)
│   ├── cron_check.py       ← скрипт ежедневной автопроверки
│   ├── config.py           ← конфигурация (читает .env)
│   ├── db.py               ← SQLite: серверы + история проверок
│   ├── ssh_worker.py       ← SSH-деплой и запуск воркеров
│   ├── report.py           ← парсинг JSONL, форматирование отчётов
│   ├── requirements.txt
│   └── data/
│       ├── sni.txt         ← список SNI для проверки (положить сюда)
│       └── sni_monitor.db  ← база данных (создаётся автоматически)
│
└── worker/                 ← воркер (деплоится на каждый удалённый сервер)
    ├── sni.py              ← скрипт проверки SNI
    ├── requirements.txt
    └── setup.sh            ← ручная установка воркера
```

## Установка в один клик

```bash
sudo bash install.sh
```

Скрипт интерактивно запросит:
- `TG_BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather)
- `TG_ADMIN_IDS` — ваш Telegram ID (узнать у [@userinfobot](https://t.me/userinfobot))
- Время ежедневной автопроверки (по умолчанию `09:00`)

Затем автоматически:
1. Установит системные зависимости (`python3`, `venv`, `cron`)
2. Скопирует файлы в `/opt/sni_monitor/`
3. Создаст виртуальное окружение и установит pip-пакеты
4. Запишет `.env` конфигурацию
5. Создаст и запустит **systemd-сервис** `sni-monitor`
6. Настроит **cron-задачу** на выбранное время
7. Установит команду **`checker`** глобально

### Неинтерактивная установка (CI/скрипты)

```bash
TG_BOT_TOKEN="123:xxx" TG_ADMIN_IDS="111222" CRON_HOUR=9 sudo bash install.sh
```

## Команда checker

После установки доступна глобальная команда:

| Команда | Описание |
|---|---|
| `checker status` | Статус бота и расписания |
| `checker start` | Запустить бота |
| `checker stop` | Остановить бота |
| `checker restart` | Перезапустить бота |
| `checker log` | Последние 50 строк логов |
| `checker log -f` | Логи в реальном времени |
| `checker run` | Запустить проверку всех серверов сейчас |
| `checker run <id>` | Проверить один сервер по ID |
| `checker cron on 09:00` | Включить ежедневную автопроверку |
| `checker cron off` | Выключить автопроверку |
| `checker cron status` | Показать расписание |

## Переменные окружения

Файл: `bot/.env`

| Переменная | Обязательная | Описание | По умолчанию |
|---|---|---|---|
| `TG_BOT_TOKEN` | ✅ | Токен бота от @BotFather | — |
| `TG_ADMIN_IDS` | ✅ | ID администраторов через запятую | — |
| `SSH_USER` | — | SSH-пользователь на воркерах | `root` |
| `SSH_PORT` | — | SSH-порт | `22` |
| `REMOTE_DIR` | — | Директория воркера на серверах | `/opt/sni_worker` |
| `SNI_CONCURRENCY` | — | Параллельность проверок | `50` |
| `HISTORY_KEEP` | — | Количество хранимых проверок | `20` |
| `CRON_HOUR` | — | Час автопроверки | `9` |
| `CRON_MINUTE` | — | Минута автопроверки | `0` |

## Управление сервисом

```bash
systemctl status  sni-monitor       # статус
systemctl restart sni-monitor       # перезапуск
journalctl -u sni-monitor -f        # логи live
crontab -l                          # расписание крона
```

Логи: `/var/log/sni_monitor/bot.log`, `/var/log/sni_monitor/cron.log`
